## 3. Missing Coverage

- **No test pins ∮ B·n dA = 0 for divergence-free B over a closed surface.** With a closed surface and divergence-free B from `BiotSavartJAX`, the **signed** flux integral should be zero to floating-point precision. This is a fundamental physics invariant. The current `target=0` quadratic-flux test pins the squared integrand magnitude, not the signed flux. Recommend a dedicated test that constructs a topological torus surface, computes `(B·n)*norm_n` summed over `nphi*ntheta`, and asserts magnitude < `1e-10` (this is also a great parity sanity check that does not depend on a CPU oracle).
- **No test pins rotational symmetry of B field under nfp transformation.** All `coil_surf_setup` and `mixed_quad_setup` fixtures use `nfp=1, stellsym=False` (lines 957, 1967, 2311, 2390). The only `nfp=5, stellsym=True` test is `TestStage2BananaBoundary::test_stage2_curve_classes_match_on_stage2_surface` (line 2796), and it compares `CurveCWSFourier` vs `CurveCWSFourierCPP`, not B-field rotational invariance. Recommend a test that for `nfp>=2`, rotates the evaluation point by `2π/nfp` and asserts `BiotSavartJAX.B(rotated_p) == R(BiotSavartJAX.B(p))` to `rtol=1e-10`.
- **No FD-vs-grad Taylor test on `SquaredFluxJAX.dJ()` directly.** Upstream `test_fluxobjective.py::test_derivatives` runs a Taylor test on the CPU `SquaredFlux`. The JAX adapter only gets CPU-vs-JAX gradient parity (which catches an absolute mismatch but not a coupled CPU+JAX bug). A 5-step `0.5**i` Taylor test on `SquaredFluxJAX` (mirroring `check_taylor_test`) would close this gap and would not require simsoptpp.
- **No "calling `set_points()` after `SquaredFluxJAX` construction raises" test.** CLAUDE.md states the pattern is INVALID, but `test_j_ignores_mutated_field_points_after_construction` (line 2121) actually documents the *current* behavior of "silently ignored". If the contract is meant to *raise* (or at least warn), there is no test asserting that. If the contract is "silently ignored is fine because the JIT closure owns the points", make that explicit in the test docstring (currently the docstring frames it as "Fixed-surface SquaredFluxJAX is driven by its immutable surface spec" — fine but does not flag the user-trap risk).
- **No Stage 2 monotone-decrease check.** The optimization-trajectory tests assert final-value parity but never assert that `JF.J()` decreases monotonically across L-BFGS-B iterations. A monotone gate (`assert all(np.diff(trajectory) <= 1e-10)`) would catch a numerical-noise regression that breaks line search but not final convergence.
- **No coverage of `target!=None` on the `normalized` and `local` definitions.** `test_fluxobjective_target_parity` (line 198) only covers `"quadratic flux"`. Both other definitions accept a target but have no parity test with non-zero target.
- **No coverage that the cached `_cached_value`/`_cached_partials` are properly invalidated when `recompute_bell` is triggered by a parent's DOF change.** `test_j_changes_after_dof_mutation` (line 1931) tests direct DOF mutation but not propagation through the Optimizable graph (e.g., a scaled current upstream).
- **No assertion that a failed JAX adjoint surfaces a non-finite gradient rather than a finite-but-wrong fallback.** CLAUDE.md notes this contract for adjoint solves; the `_raise_if_nonfinite_squared_flux_gradient` helper enforces it for `SquaredFluxJAX`, but no test injects a deliberate non-finite trace and asserts the `ObjectiveFailure`. (`test_singular_zero_current_gradients_raise_objective_failure` does cover the natural case, but not an adversarial injection.)
- **Mixed-quadrature gradient parity uses only one definition per parameterization.** `TestMixedQuadratureParity::test_gradient_parity` runs all three definitions, which is good; however, the `chunked_grouped_paths_match_cpu_on_large_point_cloud` (line 2037) test exercises the chunking but only on `B`, not on the `dJ` gradient. A gradient-parity-on-large-point-cloud test would catch a chunked-grouping bug in the VJP path that the value-only test cannot.

## 4. Tightening Playbook (P0 — top 7 changes)

1. **Replace the three TAUTOLOGICAL kernel tests** in `test_fluxobjective_jax_parity.py:211-263` with CPU-vs-JAX parity tests that drive `SquaredFluxJAX` end-to-end (one zero-area row in the surface normals, target singular configurations via real coils with current = 0). The kernel-only `_flux_kernel_value_and_grad` helper should be deleted or kept only as a private utility, not as the basis of test assertions.
2. **Add an FD-vs-grad Taylor test on `SquaredFluxJAX.dJ()`** mirroring upstream `check_taylor_test`. Run it on the `coil_surf_setup` fixture across all three definitions. This test does not require simsoptpp and would also harden the GPU lane.
3. **Add a divergence-free flux-integral test:** `assert abs(jnp.sum((B·n)*norm_n) / (nphi*ntheta)) < 1e-10` for a closed `SurfaceRZFourier` torus. This is a free physics invariant — a positive parity test that does not depend on the CPU oracle.
4. **Tighten `test_singular_zero_current_objectives_boundary_is_documented`** to assert sign-of-inf parity (`np.isposinf(cpu_j) == np.isposinf(jax_j)`) and pair with `assert isinstance(jax_j, float)` to catch a leaked JAX traced scalar.
5. **Add a per-iteration trajectory parity gate** to `test_short_run_parity` (currently only the final value is checked at `1e-3`); reuse the callback pattern from `_build_and_run_stage2`. Also tighten `assert min_len >= 5` in `test_lbfgs_trajectory_parity` to `>= 25` to make sure the gate is non-trivial.
6. **Add an nfp rotational-symmetry B-field test** with `nfp>=2`. This is a 10-line test that would have caught the historical Y/Z stellsym DOF convention bug noted in CLAUDE.md, and it complements the parity floor with a positive physics invariant.
7. **Strengthen `test_j_changes_after_dof_mutation` (line 1931)** from an inequality assertion to a centered-FD-vs-grad consistency check at the perturbation point. Same effort, much stronger signal.
