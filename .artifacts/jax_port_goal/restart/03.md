# Item 03 Restart And Serialization Check

Status: complete for CPU/JAX item 03.

## Public Compatibility Surface

`SquaredFlux` remains the legacy CPU `Optimizable` wrapper. Its
public `as_dict` / `from_dict` schema is unchanged in item 03. The
existing JSON round-trip test
`tests/objectives/test_fluxobjective.py:139-141` confirms the schema:

```python
J_str = json.dumps(SIMSON(J), cls=GSONEncoder)
J_regen = json.loads(J_str, cls=GSONDecoder)
self.assertAlmostEqual(J.J(), J_regen.J())
```

This test is preserved at the upstream commit
`1b0cc3a96063197cdbdd01559e04c25456fbe6ff` and in the repo at
`tests/objectives/test_fluxobjective.py`.

`SquaredFluxJAX` is a runtime-spec consumer; it captures the immutable
`FieldEvalSpec` and `FixedSurfaceFluxSpec` at construction and does not
participate in `as_dict` / `from_dict` serialization. The class
intentionally has no `as_dict` method: serializing the captured
device-resident specs is not on a stable JSON contract and is out of
scope for item 03 (and the broader port; the user-facing serialization
boundary is the CPU `SquaredFlux`).

## DOF-layout drift rejection

The drift-rejection contract is witnessed by these existing tests:

- `tests/objectives/test_fluxobjective_jax_parity.py:429-439` —
  `test_squaredfluxjax_rejects_field_point_mutation_after_construction`
  asserts that calling `field.set_points(...)` after constructing
  `SquaredFluxJAX` raises `RuntimeError` from
  `_raise_if_field_points_drifted` at
  `src/simsopt/objectives/fluxobjective_jax.py:279-286`.
- `tests/objectives/test_fluxobjective_jax_parity.py:441-455` —
  `test_squaredfluxjax_rejects_field_dof_layout_mutation_after_construction`
  asserts that fixing a DOF on the field after construction raises
  `RuntimeError` from `_raise_if_field_dof_layout_drifted` at
  `src/simsopt/objectives/fluxobjective_jax.py:288-297`.
- `tests/objectives/test_fluxobjective_jax_parity.py:458-467` —
  `test_squaredfluxjax_requires_native_field_contract` asserts that a
  non-native field (no `coil_dof_extraction_spec`) is rejected at
  construction with a `NotImplementedError`.

These three tests collectively define the public contract that
restart consumers rely on: a `SquaredFluxJAX` instance is valid only
for the runtime spec captured at construction. Restart-from-JSON of a
Stage 2 problem reconstructs the JAX wrapper at the same constructor
call site, not via `from_dict`.

## Surface-spec rejection

`tests/objectives/test_fluxobjective.py:207-214` —
`test_squaredfluxjax_requires_surface_spec` asserts that constructing
`SquaredFluxJAX` on a surface that does not expose `surface_spec()`
raises `NotImplementedError` with a message containing the string
`"surface_spec"`. This guards against restart pipelines that hand a
host-only surface mock into the JAX adapter.
