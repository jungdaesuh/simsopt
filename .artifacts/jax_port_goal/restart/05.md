# Item 05 Restart And Serialization Check

Status: complete for CPU/JAX item 05.

## Public Compatibility Surface

The eight curve classes in item 05 scope (`CurveXYZFourier`,
`JaxCurveXYZFourier`, `CurveRZFourier`, `CurvePlanarFourier`,
`JaxCurvePlanarFourier`, `CurveHelical`, `CurveXYZFourierSymmetries`,
`CurveCWSFourierCPP`, `CurveCWSFourier`, `CurvePerturbed`) and the
`CurveFilament` finite-build wrapper retain their existing public
`as_dict` / `from_dict` round-trip support through the GSON encoder /
decoder used by SIMSOPT. Item 05 does not modify any serialized layout, DOF
ordering, or class constructor signature.

The immutable JAX specs (`CurveXYZFourierSpec`, `CurveRZFourierSpec`,
`CurvePlanarFourierSpec`, `CurveHelicalSpec`, `CurveCWSFourierRZSpec`,
`CurvePerturbedSpec`, `CurveFilamentSpec`) are deterministic functions of the
host-side curve state. They are rebuildable on demand from a deserialized
curve via `curve_spec_from_curve(curve)`; no separate restart payload exists
or is changed.

## Checks

- Existing `tests/geo/test_curve.py` covers serialized curve reload for
  every curve class listed in `Testing.curvetypes` (XYZ Fourier, JaxXYZ
  Fourier, RZ Fourier, Planar Fourier, Jax Planar Fourier, Helical, three
  variants of CurveXYZFourierSymmetries, and a HelicalInitx0 case). Each
  variant is run through `Testing.subtest_serialization` which calls
  `GSONEncoder` then `GSONDecoder` and asserts identical DOFs, geometry,
  curve length, and Taylor-test convergence on derivatives.
- `tests/geo/test_curve.py::test_curvexyzsymmetries_raisesexception` covers
  the gcd(nfp, ntor) constructor guard, which is invariant under
  serialization.
- `tests/integration/test_single_stage_jax_cpu_reference.py::TestBiotSavart
  JAXReference::test_curveperturbed_to_spec_preserves_sample_derivatives`
  and `::test_curvefilament_to_spec_preserves_frame_and_offsets` verify
  that `to_spec()` reads the same `sample` and `framedcurve` state used by
  the legacy host wrappers, which is the relevant restart-compatibility
  surface for those two wrapper classes.
- Item 05's new tests in `tests/geo/test_curve_item05_closeout.py` use
  `curve.gamma()` (CPU oracle) and `curve_geometry_from_dofs(spec, dofs)`
  (JAX path). They do not introduce or change any serialized artifact and
  do not require any restart migration.

## Result

No restart artifact requires migration for item 05.
