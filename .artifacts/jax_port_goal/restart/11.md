# Item 11 Restart And Serialization Check

## Public Compatibility Surface

Item 11 introduces a **new** pure-JAX module
``src/simsopt/jax_core/analytic_fields.py`` carrying:

- ``DommaschkSpec`` (frozen dataclass; not part of the public
  ``Optimizable`` graph).
- ``ReimanSpec`` (frozen dataclass; not part of the public
  ``Optimizable`` graph).
- Pure functions ``dommaschk_B``, ``dommaschk_dB``, ``reiman_B``,
  ``reiman_dB`` that operate on those specs plus a points array.

The public Python wrappers
``simsopt.field.magneticfieldclasses.Dommaschk`` and
``simsopt.field.magneticfieldclasses.Reiman`` are **unchanged** by
item 11. They continue to call the C++ ``sopp.DommaschkB`` /
``sopp.DommaschkdB`` / ``sopp.ReimanB`` / ``sopp.ReimandB`` kernels
and continue to support their existing ``as_dict`` / ``from_dict``
serialization roundtrip:

- ``Dommaschk.as_dict`` / ``Dommaschk.from_dict`` (lines 787-799 of
  ``magneticfieldclasses.py``).
- ``Reiman.as_dict`` / ``Reiman.from_dict`` (lines 833-844 of
  ``magneticfieldclasses.py``).

Existing serialization tests in
``tests/field/test_magneticfields.py::test_Dommaschk`` (lines
657-659 and 703-706) and ``::test_Reiman`` (lines 975-978) continue
to round-trip the C++-backed wrappers without change.

## Checks

- The new ``DommaschkSpec`` / ``ReimanSpec`` dataclasses are frozen
  and do not implement ``as_dict`` / ``from_dict``; they are
  runtime-only JAX containers and are reconstructed on each call
  from Python integer tuples and JAX arrays. No new serialization
  artifact is introduced.
- The new ``tests/jax_core/test_analytic_fields_item11.py`` parity
  tests use direct C++ cross-oracle and closed-form fixtures; they
  do not exercise serialization.

## When Item 15 Lands

Item 15 (the Python wrapper rewrite) will swap the
``magneticfieldclasses.Dommaschk`` and ``Reiman``
``_B_impl`` / ``_dB_by_dX_impl`` methods to call the new JAX
kernels. At that point the existing ``as_dict`` / ``from_dict``
schema must continue to round-trip; the wrapper-level
serialization tests already cover that contract.

No restart-format migration is required for item 11.
