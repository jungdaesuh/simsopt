## Public-Facing Artifact Surface

The in-scope item-04 modules touch SIMSOPT public surfaces in two ways:

1. **CPU `BoozerSurface` parity oracle** (`src/simsopt/geo/boozersurface.py`)
   — unchanged. Existing serialization tests (`tests/geo/test_boozersurface.py::test_boozer_serialization`)
   exercise `as_dict`/`from_dict` round-trip via the upstream-equivalent
   `simsopt._core.json` machinery. This item does NOT touch
   `BoozerSurface.as_dict`/`from_dict`.

2. **JAX adapters** (`BoozerSurfaceJAX`, `BoozerResidualJAX`, `IotasJAX`,
   `MajorRadiusJAX`, `NonQuasiSymmetricRatioJAX`) — these inherit from
   `Optimizable` but are not configured for SIMSOPT JSON round-trip in
   the public release surface. They are constructed at runtime from
   CPU `BoozerSurface` plus `BiotSavartJAX` and do not need to survive
   JSON serialization. Restart compatibility is provided by reloading
   the upstream CPU artifact (e.g. `bs.json`) and reconstructing the
   JAX adapter in memory.

## Runtime Spec Compatibility

- `_BoozerSurfaceRuntimeState`, `_BoozerSolvedRuntimeState`,
  `_BoozerAdjointRuntimeState`, `_BoozerPenaltyVectorizedInputs`, and
  `_BoozerLSGroupedVJPSnapshot` are immutable dataclasses. They are
  rebuilt deterministically from the surface and field state at every
  construction; no on-disk serialization.
- `make_traceable_objective_runtime_bundle()` caches bundles in process
  memory only; no on-disk cache. CLAUDE.md note: rebuild the bundle
  after changing inputs; do not mutate captured objects and expect an
  existing cached bundle to retarget itself.
