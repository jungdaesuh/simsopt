## Oracle Contract

For each curve class with a spec path, the public CPU oracle is the curve's
own `.gamma()` (and `.gammadash()` where exercised). The parity test compares
`curve.gamma()` (legacy CPU `Optimizable` evaluation) against the JAX-side
`curve_geometry_from_dofs(spec, dofs)["gamma"]` returned through
`curve_spec_from_curve(curve)`. This is a fixed-state scalar / vector parity
contract; no optimizer trajectory is claimed.

For `CurveXYZFourierSymmetries`, the JAX-spec path is not available on the
parent commit. The pure-JAX kernel still runs through the legacy `JaxCurve`
hot path (`gamma_pure` -> `gamma_jax`), so `curve.gamma()` is itself a JAX
evaluation. The parity row exists as a documented `not_applicable_pure_jax`
classification with a `pytest.skip(...)` and an architecture-blocker docstring.

Upstream SIMSOPT (`/Users/suhjungdae/code/opensource/simsopt/src/simsopt/geo/`)
at SHA `1b0cc3a96063197cdbdd01559e04c25456fbe6ff` does not own the
`curve_spec_from_curve` adapter (this is a `simsopt-jax`-local helper).
Upstream `CurveXYZFourierSymmetries` is identical to the in-tree class up to
formatting and remains a pure JaxCurve with no immutable spec. The upstream
oracle is therefore the in-process CPU evaluation `curve.gamma()`.
