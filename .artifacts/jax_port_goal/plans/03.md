## Oracle Contract

`SquaredFlux.J()` is the CPU C++ oracle. `sopp.integral_BdotN` returns the
identical scalar for the 3 definitions (see `integral_BdotN.cpp:93-119`).
`SquaredFlux.dJ()` is the gradient oracle (analytic dJ/dB block × `B_vjp`).

The parity contract is `fixed_state_scalar_and_fixed_state_gradient_vjp`.
For item 03 the new closeout test asserts only the scalar `J()` parity
since the JAX gradient path is already covered by the existing
`test_fluxobjective_jax_parity.py` and `test_stage2_jax.py` fixtures (see
the coverage matrix).

## New Closeout Test

`tests/objectives/test_fluxobjective_jax_item03_closeout.py` adds one
parametrized test that closes the production-scale × all-3-definitions ×
strict-transfer × stellsym=True dimension that is not bundled into a
single existing fixture. Each parametrized invocation:

- Builds a LandremanPaul-style production-scale fixture: `nfp=2`,
  `stellsym=True`, `ncoils=4` base coils (8 total after symmetry),
  `nphi=16, ntheta=8`.
- Compares `SquaredFluxJAX.J()` against `SquaredFlux.J()` (CPU C++
  oracle via `sopp.integral_BdotN`).
- Asserts the parity at `direct_kernel` lane tolerances imported from
  `benchmarks.validation_ladder_contract.parity_ladder_tolerances`.
- Runs under both relaxed and `SIMSOPT_JAX_TRANSFER_GUARD=disallow`
  invocations (process-level env), since the JAX code path uses an
  immutable spec captured at construction and `host_scalar` / `host_array`
  helpers at the public boundary.
