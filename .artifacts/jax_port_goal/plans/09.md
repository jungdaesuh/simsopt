## Source Audit

Upstream audit SHA:
`1b0cc3a96063197cdbdd01559e04c25456fbe6ff`.

- Upstream `src/simsopt/field/force.py` defines the same kernel signatures and
  six `Optimizable` wrappers. Upstream constructs `BiotSavart`-light kernels in
  pure JAX without the fork-only `_SharedCoilState` pre-compute layer.
- Fork additions are purely host-side caching wrappers around the same JAX
  graph: `_SharedCoilState` packs per-coil `gamma`, `gammadash`, optional
  `gammadashdash`, and `current` arrays; `_CoilStateGroupCache` stacks them
  across coils for the JAX entrypoints. The compiled compute does not differ
  from upstream once the inputs are packed.
- Module-level lifted `jit + grad` entrypoints replace upstream's per-class
  `self.J_jax = jit(...)` pattern. Each class binds `self.J_jax` and
  `self.dJ_jax` to the same compiled functions so cross-instance compile-cache
  hits are deterministic.
- The objective wrappers route through `_J_args` -> `_cached_objective_args` ->
  `_CoilStateGroupCache.arrays` so a coil group state is reused until a
  dependent `Optimizable` invalidates it via `recompute_bell`.

## Oracle Contract

All six wrappers and all kernels in scope already match the upstream pure JAX
implementation at the audit SHA. The hand-derived oracle in
`tests/field/test_selffieldforces.py::TestForces::test_force_objectives`
(lines 1191-1539) compares each wrapper value against a NumPy-side restatement
of the same math at `ncoils=4`, `nfp=3`, `numquadpoints=15*order=90` after
`coils_via_symmetries` expansion (`24` coils). Tolerance there is the existing
inline `rtol=1e-6`.

parity assertion lives on the same documented contract as the rest of the
JAX-port closeout work. No inline numeric tolerance is added by the new test.
