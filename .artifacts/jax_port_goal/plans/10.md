## Oracle Contract

`fixed_gradient_vjp` is not the contract for this item. The contract is
`fixed_scalar` chained parity at the production-scale floor:

- For each definition in `{quadratic flux, normalized, local}` and each
  symmetry mode in `{stellsym=False, stellsym=True}`:
    - Build a coil set with ncoils=4 base coils (8 after stellsym, 4
      without).
    - Build a `SurfaceRZFourier` with `nphi=16`, `ntheta=8`.
    - Evaluate `B_cpu = BiotSavart(coils).set_points(...).B()` and
      `B_jax  = BiotSavartJAX(coils).set_points(...).B()`.
    - Reshape both to `(nphi, ntheta, 3)`.
    - Build a `target = (B_cpu * n) / |n|` style nontrivial target so the
      `normalized` / `local` denominators are exercised. The target is
      also evaluated at the C++ oracle's B, so both sides see the same
      target.
    - Compare:
        - `B_jax` to `B_cpu` at the `direct-kernel` lane tolerance.
        - `sopp.integral_BdotN(B_jax_host, target, normal_host, def)`
          to `sopp.integral_BdotN(B_cpu, target, normal_host, def)` —
          this anchors integral_BdotN to the C++ surface integral oracle
          using whichever B array the lane provides.
        - `host_scalar(integral_BdotN(B_jax, target, normal, def))` to
          `sopp.integral_BdotN(B_cpu, target, normal_host, def)` — this
          is the full chained parity claim.

Tolerances come from
`benchmarks.validation_ladder_contract.parity_ladder_tolerances("direct_kernel")`
which sets `rtol=1e-10`, `atol=1e-12` (same-state requirement, vector
parity required).
