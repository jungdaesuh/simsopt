## Source Audit

Every non-distance public class in this module is already JAX-native in
the parent commit `a9da18fac`. Each public `dJ()` consumes a pure-JAX
`grad`-based helper and projects the host gradient back through the
existing curve VJP API. `LinkingNumber.J()` is the only kernel still
backed by C++ (`sopp.compute_linking_number`); its `dJ()` returns a
zero `Derivative({})` by construction.

Existing pre-item-07 FD-Taylor parity tests in
`tests/geo/test_curve_objectives.py` cover the differentiable modes of
`CurveLength`, `LpCurveCurvature`, `LpCurveCurvatureBarrier`,
`LpCurveTorsion`, `ArclengthVariation`, and `MeanSquaredCurvature`.
Upstream HSS testing for `LinkingNumber` is also mirrored locally
(`tests/geo/test_curve_objectives.py:1057`).

The two coverage gaps closed by this item are:

1. `FramedCurveTwist` did not have a finite-difference Taylor parity
   test for the `f="lp"` differentiable mode. The existing
   `test_framed_curve_twist_reuses_shared_jit_kernels` test only
   exercised the JIT-cache hygiene boundary.
2. `FramedCurveTwist` did not have a contract test for the
   non-differentiable modes `f in {net, range, max}` confirming that
   `J()` is finite and `dJ()` returns a zero-`Derivative` block (the
   source contract at `curveobjectives.py:1375` is unconditional).
3. `LinkingNumber` lacked a production-scale fixture at the documented
   `ncoils>=4` floor. The existing `test_linking_number` test fixes
   `ncoils=2`.

## Oracle Contract

Upstream SIMSOPT audit SHA used here:
`1b0cc3a96063197cdbdd01559e04c25456fbe6ff`.

- `CurveLength`, `LpCurveCurvature`, `LpCurveTorsion`,
  `ArclengthVariation`, `MeanSquaredCurvature`, and `LinkingNumber` are
  documented upstream public objectives. Their CPU oracles are mirrored
  in the local `tests/geo/test_curve_objectives.py` FD-Taylor and
  linking-number value tests.
- `LpCurveCurvatureBarrier` is port-only; the oracle is the hand-derived
  log-barrier scalar:
  `J = mean(-log(1 - kappa/threshold) * |gammadash|)`, infeasible when
  any `kappa >= threshold`.
- `FramedCurveTwist` is port-only; the oracle is the hand-derived
  composed twist:
  - `frametwist_pure(...)` discrete trapezoid integral of the rotation
    angle profile.
  - `frametwist_lp_pure(profile, gammadash, p) = (mean(profile**p *
    arc_length) / mean(arc_length))**(1/p)`.
  - The `dJ()` contract for non-lp modes is an unconditional
    `Derivative({})` (source: `curveobjectives.py:1375`).

## Implementation Changes

No source-code change is required by item 07. The non-distance objectives
were already JAX-native at the parent commit `a9da18fac`. This item adds
targeted tests under
`tests/geo/test_curveobjectives_item07_closeout.py`.

The new file covers:

1. `test_framed_curve_twist_lp_taylor_value_and_gradient`: builds a
   small `FramedCurveCentroid(CurveXYZFourier, FrameRotation)` fixture
   and checks FD-Taylor consistency of `J()` and `dJ()` over the
   union of curve and rotation dofs in `f="lp"` mode at five halving
   step sizes.
2. `test_framed_curve_twist_non_lp_modes_return_zero_derivative`:
   parametrized over `f in {net, range, max}`, asserts `J()` is finite
   and `dJ()` is a zero-`Derivative` block of the correct dof count.
3. `test_linking_number_production_scale_ncoils_four`: builds an
   `ncoils=4`, `nfp=2`, `stellsym=True` ring of equally-spaced
   `CurveXYZFourier` coils via the project's
   `create_equally_spaced_curves` helper and asserts `LinkingNumber.J()`
   is exactly `0` (matching the well-known unlinked planar TF set) and
   `LinkingNumber.dJ()` is a zero-`Derivative` block.
4. `test_linking_number_production_scale_ncoils_four_strict_transfer_guard`:
   same fixture, executed under `jax.transfer_guard("disallow")` to
   prove the `J()` boundary stays strict.
