Public classes (audited at HEAD `a9da18facfda9a51f14534ea9d06e76803213db0`):

- `NormalField(Optimizable)` (line 20). `__init__` takes `nfp`, `stellsym`,
  `mpol`, `ntor`, optional `vns` / `vnc` 2D arrays, and an optional
  `SurfaceRZFourier`. DOF packing is computed via `get_dofs` (line 167) and
  `get_index_in_dofs` (line 222). The stellsym DOF count is
  `ntor + mpol * (2 * ntor + 1)`; the non-stellsym count is
  `2 * (ntor + mpol * (2 * ntor + 1)) + 1`. The Fourier transform of `vns` /
  `vnc` to a real-space normal field uses
  `SurfaceRZFourier.inverse_fourier_transform_scalar` with the SPEC-style
  `(2 * pi) ** 2` normalization (line 517), divided by the host
  `np.linalg.norm(surface.normal(), axis=-1)` to convert the unnormalized
  `B . normal` integrand into the real-space `B . unit-normal`.
- `CoilNormalField(NormalField)` (line 522) inherits the Fourier helpers but
  derives `vns` / `vnc` from a `CoilSet` parent. The cached
  `np.sum(coilset.bs.B().reshape((nphi, ntheta, 3)) * surface.normal() * -1,
  axis=2)` reduction (lines 576-577 and 589-590) feeds
  `surface.fourier_transform_scalar(..., normalization=(2 * pi) ** 2,
  stellsym=...)` to recover the SPEC-convention Vns/Vnc harmonics. The
  Optimizable `recompute_bell` (line 639) clears `_vns` / `_vnc` so the next
  property access re-evaluates against the current coil DOFs.

The module is pure NumPy at runtime; no `jax`, `jaxlib`, `simsoptpp`, or
`Derivative` projection is imported by `normal_field.py`. The Fourier
transform helpers it consumes
(`SurfaceRZFourier.fourier_transform_scalar` at
`src/simsopt/geo/surfacerzfourier.py:2169` and
`SurfaceRZFourier.inverse_fourier_transform_scalar` at line 2269) are also
pure-NumPy methods on the public `SurfaceRZFourier` class. They are not
listed in the goal-prompt skip list and are not C++ binding methods — they
are Python methods defined alongside the C++ binding surface area.

`coilset.bs` is the CPU `BiotSavart` parity oracle (per the goal-prompt
skip list, `src/simsopt/field/biotsavart.py` is the parity oracle and may
not be modified). `BiotSavartJAX` is not invoked by `CoilNormalField` at
HEAD; the JAX-native path is reachable via `flux_penalty` and the Stage-2
objective stack, not via the Vns / Vnc cache itself.

## Oracle Contract

`fixed_scalar`. The closeout asserts three bit-tight invariants at the
`direct_kernel` parity-ladder lane (`rtol=1e-10`, `atol=1e-12`):

- `Fourier-pair identity`: for any allowed `(Vns, Vnc)` harmonic array of
  shape `(mpol + 1, 2 * ntor + 1)` populated through the
  upstream-canonical sine / cosine mode-set (`m = 0, n <= 0` excluded
  from sine; `m = 0, n < 0` excluded from cosine), the round trip
  `fourier_transform_scalar(inverse_fourier_transform_scalar(Vns, Vnc))`
  reproduces `(Vns, Vnc)` bit-tight. Hand-derived from the upstream
  docstring at `surfacerzfourier.py:2249-2253` ("The next 2 lines ensure
  inverse Fourier transform(Fourier transform) = identity").
- `CoilNormalField cache parity`: `cnf.vns` and `cnf.vnc` reproduce a
  hand-rolled CPU oracle computed as
  `surface.fourier_transform_scalar(np.sum(bs.B().reshape((nphi, ntheta,
  3)) * surface.normal() * -1, axis=2), normalization=(2*pi)**2,
  stellsym=...)`. The oracle uses the same `BiotSavart` instance and
  the same `SurfaceRZFourier` instance, so this is a same-state
  reduction-vs-direct-numpy parity claim and lives on the
  `direct_kernel` lane.
- `Cache invalidation parity`: after changing a `CoilSet` DOF and
  triggering `recompute_bell`, the next `cnf.vns` access reproduces the
  hand-rolled oracle for the new DOF. This catches a regression where
  the `_vns` / `_vnc` cache fails to invalidate.

A negative control verifies that dropping the `-1` sign convention from
the `CoilNormalField` reduction produces a tolerance-busting Vns
deviation. Without the negative control, a sign flip in the reduction
would silently pass the same-state parity claim.
