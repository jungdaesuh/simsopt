## Oracle Contract

CurrentPenalty has no matching upstream hiddenSymmetries public class at
upstream audit SHA `1b0cc3a96063197cdbdd01559e04c25456fbe6ff`. The oracle is
the hand-derived scalar contract
`max(abs(I) - threshold, 0)^2` and derivative
`2 * (abs(I) - threshold) * sign(I)` above threshold, projected through
`Current.vjp`, `ScaledCurrent.vjp`, and `CurrentSum.vjp`.

For distance wrappers, upstream SIMSOPT source and official docs establish the
public API and C++ oracle behavior:

- `CurveCurveDistance` computes an inter-coil distance penalty and supports
  `J`, `dJ`, `compute_candidates`, `shortest_distance`, and
  `shortest_distance_among_candidates`.
- `CurveSurfaceDistance` computes a coil-to-surface distance penalty.
- Upstream C++ candidate culling uses lower-triangle / `num_basecurves`
  semantics for curve-curve pairs and rectangular cross-collection semantics
  for curve-surface pairs.
