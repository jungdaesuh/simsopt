### Algorithm — kernel structure

The CPU oracle (`surfacehenneberg.py:588-739`) decomposes into three
real-space stages:

1. **1D φ-axis modes**: `R0H(φ), Z0H(φ), b(φ)` and their first derivatives
   from `cos/sin(nfp·n·φ)` (with `Z0nH[0] = 0` enforced by mask).
2. **2D ρ(θ, φ) mode sum**: `Σ_{m,n} ρ_{m,n} cos(m·θ + (nfp·n - α)·φ)`
   over a `(mmax+1, 2·nmax+1)` mode grid. The host-class `(m=0, n<=0)`
   cells are masked. Derivatives wrt φ and θ derive from the same
   tensor by per-mode scaling of `sin(·)` contributions.
3. **Position composition**: `R = R0H + ρ cos(αφ) - ζ sin(αφ)`,
   `Z = Z0H + ρ sin(αφ) + ζ cos(αφ)`, `γ = (R cosφ, R sinφ, Z)`. The
   `gammadash1` chain rule re-uses the φ-axis 1D derivatives plus the
   tensor `∂ρ/∂φ`; `gammadash2` re-uses `∂ρ/∂θ` and the cheap
   `∂ζ/∂θ = b cos(θ - αφ)`.

The CPU oracle uses `np.meshgrid(phi_1D, theta_1D)` (`xy` indexing,
shape `(ntheta, nphi)`) then transposes at the end. The JAX kernels
use `jnp.meshgrid(..., indexing='ij')` (shape `(nphi, ntheta)`)
directly, eliminating the transpose. The final output buffer matches
the CPU `surface.gamma()` axis order `(nphi, ntheta, 3)`.

### Strict-mode hygiene

All Python scalars that enter the JAX trace go through
`jnp.asarray(value, dtype=...)` so the kernels pass under
`jax.transfer_guard('disallow')`. The `_TWO_PI`, `α`, and `nfp`
constants are materialised as device-resident float64 scalars at the
boundary; index arrays (`m_grid`, `n_grid`, `valid_mask`) are built
host-side, cast via `as_jax_float64`, and baked into the trace as
constants (they depend only on `(mmax, nmax)`, which are meta_fields).

## Residual risk

- Second-order geometry (`gammadash1dash1`, `gammadash1dash2`,
  `gammadash2dash2`) is **out of scope**: the CPU `SurfaceHenneberg`
  class itself does not implement `gammadash*_impl` second-derivative
  kernels. A future agent that needs them must either upstream a CPU
  implementation first (then port it here) or route the higher
  derivatives through `SurfaceXYZTensorFourier` after a host-side
  conversion.
- `from_RZFourier` (real-space → Henneberg least-squares fit) remains
  host-only; it is an analysis-time utility that uses
  `scipy.optimize.minimize_scalar` and `scipy.interpolate.interp1d`,
  not a hot path. Not in scope here.
